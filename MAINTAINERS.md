## Responsibilities

All maintainers are responsible for:

- **Code & Data Review**: Review all pull requests and data submissions before merging
- **Data Quality**: Validate new data entries according to RILEM TC 306-CFR standards
- **Documentation**: Keep documentation updated
- **Communication**: Respond to issues and questions from contributors
- **Releases**: Manage versioning, tags, and releases on both GitHub and Zenodo
- **License Compliance**: Ensure all contributions comply with CC BY 4.0 license

---

## Review Process

When a pull request is submitted:

1. A maintainer will review the changes within 1-2 weeks
2. Feedback or approval will be provided
3. Once approved, the maintainer will merge the changes into the main branch
4. The maintainer will coordinate with Zenodo for version updates if needed

---

## Becoming a Maintainer

Contributors who have made significant, consistent contributions to the database can apply for maintainer status. Please contact the current maintainers to discuss opportunities.
